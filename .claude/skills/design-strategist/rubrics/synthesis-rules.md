# Synthesis Rules

The goal is to turn research into a strong design direction.

Always:
- start from the user job and feature goal
- use PRD constraints as real constraints, not suggestions
- use competitor findings to learn patterns, not to copy blindly
- identify the best default flow before exploring alternatives
- distinguish between:
  - required flow steps
  - optional enhancements
  - unresolved questions
- optimize for clarity, task completion, and good product judgment

Do not:
- jump straight to visual design details
- overfit to competitor patterns
- create extra complexity without a strong reason
- invent unsupported product behavior
