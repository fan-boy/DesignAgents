# Output Format

Always structure the response as:

## Feature Context
Short summary of the feature and user goal.

## Design Goal
What the design needs to achieve.

## Key Constraints
Business, technical, UX, trust, or system constraints.

## Design Directions
2-3 possible directions if useful, otherwise 1 strong recommended direction.

## Recommended Strategy
What approach to take and why.

## User Flow
Step-by-step flow with decision points and edge cases.

## Wireframe Plan
Suggested low-fi frame structure by screen or step.

## Risks and Tradeoffs
What to watch out for.

## Next Design Actions
What the designer should do next.

## Files Saved
List the generated files.
