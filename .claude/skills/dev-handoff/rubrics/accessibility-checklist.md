# Accessibility Checklist

Include accessibility requirements that engineers can act on.

Check for:
- semantic labels for inputs and actions
- clear error messaging
- visible focus states
- logical keyboard order
- keyboard access for all actions
- screen-reader clarity for icons and status changes
- touch targets for mobile where relevant
- non-color-dependent meaning
- guidance for status, warning, and error announcements
- support for recovery and comprehension, not just compliance
