# Handoff Sections

Every handoff should include:

1. Feature Summary
- What the feature does
- Who it is for
- What outcome it supports

2. Scope
- What is included
- What is explicitly out of scope

3. Flow Coverage
- Entry points
- Key steps
- Exit states

4. Screen / Component Notes
- What appears on each screen
- What the user can do
- What the system does

5. Interaction Details
- validation
- confirmations
- progressive disclosure
- disabled states
- system feedback

6. States
- empty
- loading
- success
- error
- partial success
- permission / access states
- destructive / warning states

7. Design System Guidance
- components to reuse
- variants to use
- spacing / hierarchy constraints when relevant

8. Accessibility Requirements
- keyboard behavior
- labels
- focus order
- status announcements
- contrast or icon clarity notes
- touch target considerations when relevant

9. Analytics / Instrumentation
- key events
- success markers
- error events

10. Acceptance Criteria
- what must be true for implementation to match intent

11. Open Questions
- unresolved items requiring PM, design, or engineering decisions
