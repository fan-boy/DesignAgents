# Implementation Rules

The handoff should:
- translate design intent into build-ready guidance
- be specific enough to reduce ambiguity
- avoid repeating unnecessary visual details already obvious in Figma
- call out behavior, logic, state changes, and constraints
- separate confirmed behavior from open questions
- distinguish required implementation details from design suggestions

Do not:
- invent engineering architecture
- promise unsupported backend behavior
- over-spec UI styling that belongs in the design file
- hide uncertainty
