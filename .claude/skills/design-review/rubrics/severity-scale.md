# Severity Scale

Use when logging heuristic violations in design reviews or Jira.

| Severity | Label | Description | Action |
|---|---|---|---|
| S0 | Blocker | Prevents task completion or causes data loss | Must fix before launch |
| S1 | Critical | Causes significant confusion, errors, or trust failure | Must fix before launch |
| S2 | Major | Degrades experience meaningfully; workaround exists | Fix in current cycle |
| S3 | Minor | Small inconsistency or missed polish | Fix if time allows |
| S4 | Suggestion | Enhancement with no current violation | Backlog for consideration |
